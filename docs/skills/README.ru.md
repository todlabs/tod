# Навыки (Skills)

Навыки — это переиспользуемые наборы инструкций, которые расширяют возможности TOD. Они позволяют создавать пользовательские рабочие процессы и промпты, которые можно вызывать простой командой со слешем.

## Что такое навыки?

Навыки — это как макросы для разработки с помощью ИИ. Вместо того чтобы каждый раз вводить одни и те же инструкции, вы создаете навык один раз и вызываете его через `/название-навыка`.

## Как работают навыки

1. **Создайте навык** — Напишите инструкции в файле `SKILL.md`
2. **Сохраните** — Поместите в `~/.tod/skills/` (глобально) или `.tod/skills/` (для проекта)
3. **Вызовите** — Введите `/название-навыка` в TOD
4. **TOD выполнит** — Инструкции навыка направят ответ ИИ

## Быстрый старт

### Создание первого навыка

1. Создайте директорию навыка:
   ```bash
   mkdir -p ~/.tod/skills/moj-navik
   ```

2. Создайте файл `SKILL.md`:
   ```markdown
   # Мой навык
   
   Краткое описание того, что делает этот навык.
   
   ## Инструкции
   
   Когда этот навык вызван, выполни следующие шаги:
   1. Шаг первый
   2. Шаг второй
   3. Шаг третий
   
   ## Примеры
   
   ### Пример 1
   Пользователь: /moj-navik
   ИИ: [Ожидаемое поведение]
   ```

3. Используйте в TOD:
   ```
   > /moj-navik
   ```

## Формат навыка

Навыки — это markdown-файлы со следующей структурой:

```markdown
# Название навыка

Краткое описание (отображается в списке /skills)

## Инструкции

Подробные инструкции для ИИ.

## Примеры

### Пример 1
Ввод: ввод пользователя
Вывод: ожидаемый вывод
```

## Расположение навыков

### Глобальные навыки
Хранятся в `~/.tod/skills/` и доступны во всех проектах:
```
~/.tod/skills/
  ├── commit/
  │   └── SKILL.md
  ├── review/
  │   └── SKILL.md
  └── test/
      └── SKILL.md
```

### Проектные навыки
Хранятся в `.tod/skills/` в корне вашего проекта:
```
my-project/
  ├── .tod/
  │   └── skills/
  │       ├── deploy/
  │       │   └── SKILL.md
  │       └── release/
  │           └── SKILL.md
  └── src/
```

**Приоритет**: Проектные навыки переопределяют глобальные с тем же именем.

## Встроенные навыки

TOD поставляется с примерами навыков:

### skill-creator
Помогает создавать новые навыки:
```
> /skill-creator
```

### commit
Создает сообщения коммитов в conventional commits формате:
```
> /commit
```

## Команды

| Команда | Описание |
|---------|----------|
| `/skills` | Показать все доступные навыки |
| `/название-навыка` | Вызвать навык |

## Лучшие практики

1. **Делайте навыки сфокусированными** — Один навык = одна задача
2. **Используйте понятные имена** — строчные буквы с дефисами: `code-review`, `git-commit`
3. **Пишите хорошие описания** — Первый абзац становится кратким описанием навыка
4. **Включайте примеры** — Показывайте ожидаемые вводы и выводы
5. **Используйте контроль версий** — Коммитьте проектные навыки в git
6. **Делитесь глобально** — Помещайте переиспользуемые навыки в `~/.tod/skills/`

## Создание навыков с помощью ИИ

Используйте встроенный навык `skill-creator`:

```
> /skill-creator
ИИ: Как вы хотите назвать ваш навык?
Пользователь: deploy-vercel
ИИ: Что должен делать этот навык?
Пользователь: Деплоить текущий проект на Vercel
ИИ: [Создает файл навыка]
```

## Расширенные возможности

### Аргументы навыков
Навыки могут принимать аргументы после команды:
```
> /commit исправить баг аутентификации
```

### Контекст навыка
Навыки имеют доступ к:
- Содержимому текущей директории
- Статусу git
- Переменным окружения
- Контексту предыдущего разговора

## Устранение неполадок

**Навык не найден?**
- Проверьте имя навыка (чувствительно к регистру)
- Убедитесь, что файл называется `SKILL.md`
- Выполните `/skills` чтобы увидеть доступные навыки

**Навык не работает?**
- Проверьте синтаксис markdown
- Убедитесь, что инструкции ясны и конкретны
- Смотрите примеры навыков для справки
